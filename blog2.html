Async vs defer

So before knowing anything we must know how a browser parses a html document.

The browser parses any html document from top to bottom.But whenever it encounters a img,link tag it sends out a request for that resources and continues parsing .The important thing to note is that the browser does not stop parsingthe html document to get the img source .

However whenever the browser encouters the script tag it stops parsing the html file and downloads the javascript and executes it.

These it the reason why the script tag is place at the bottom of the html file, so that there is no delay parsing the html file.

async

Syntax: <script async src="script.js" > </script>

The async attribute when applied to the script tag will make the script tag behave like a img tag, that is when the browser encounters a script it will not stop parsing just like when it encounters a img tag and sends a request to download the javascript.

However, when the javascript file is downloaded and the browser is not completed parsing it will stop parsing and execute the javascript file and then continue parsing the html file.

defer

Syntax: <script defer src="script.js" > </script>

When defer is applied the browser will download the javascript file in the background, but unlike async it wont stop parsing when the javascript resource are downloaded, rather it will complete the parsing of the html document and then execute the javascript file.

Also the javascriptfile would be executed in order they are defined.
